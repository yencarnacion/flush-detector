Goal
Create a live, web-based watchlist scanner for a shared intraday “flush detector” and “flush score” for mean-reversion candidates. This is NOT the full FB / RB / V setup detector yet. This is the shared stage-1 detector only: it should detect stretched/climactic downside flushes and rank them with a 0-100 flush score.

- `.env` file for API key
- `config.yaml`

- configurable server port
- alert cards
- recent news and SEC filings on the alert cards
- WAV alert sounds
- similar repo layout and operational simplicity

High-level product spec
Build `flush-detector` as a complete repo that:
1. Loads config from:
   - `.env`
   - `config.yaml`
   - `watchlist.yaml`
2. Connects to Massive market data for a watchlist of stock tickers.
3. Maintains live 1-minute bars per symbol.
4. Calculates the shared flush score on completed 1-minute bars using only information available up to that bar.
5. Raises an alert when a symbol enters a configurable flush-score threshold during the active time window.
6. Pushes alerts live to a browser UI over websocket.
7. Shows alert cards with:

   - recent SEC filings

10. Supports watchlist reload without restarting, like alertcat-style operational workflow.

Core detector requirements
This project is only the shared flush detector. It should detect the common stretch/climax state before any later FB/RB/V confirmation logic.

Important implementation rules:
- Calculate only from completed 1-minute bars.
- No lookahead.
- Everything must be computable from data available at that moment.
- Use Eastern Time.
- Reset session VWAP at the session start.
- Default active window: 09:40 to 15:30 ET.
- Make these time settings configurable.

Flush score
Implement the flush score exactly as follows unless there is a very strong technical reason to preserve behavior while refactoring the code structure.

Definitions
For each completed 1-minute bar `t`:

Helper:
clip(x, lo, hi) = max(lo, min(hi, x))

Use:
- current bar OHLCV = `bar_t`
- `price_ref` = `bar_t.low`
- `close_t` = `bar_t.close`

1) Drop from prior 30-minute high
`recent_high_30` =
  max(high of the PRIOR 30 completed 1-minute bars, excluding current bar)

If fewer than 30 prior bars exist, use all available prior bars, but do not allow alerts until `min_bars_before_alerts` is satisfied.

`drop_from_prior_30m_high_pct` =
  max(0, (recent_high_30 - price_ref) / recent_high_30 * 100)

2) Distance below session VWAP
Compute session VWAP from session start through current bar using bar typical price:
`typical_price = (high + low + close) / 3`

Running VWAP:
`session_vwap = sum(typical_price * volume) / sum(volume)`

Then:
`distance_below_vwap_pct` =
  max(0, (session_vwap - price_ref) / session_vwap * 100)

3) 5-minute downside rate of change
`close_5 = close from 5 completed bars ago`
`roc_5m_pct = max(0, (close_5 - close_t) / close_5 * 100)`

4) 10-minute downside rate of change
`close_10 = close from 10 completed bars ago`
`roc_10m_pct = max(0, (close_10 - close_t) / close_10 * 100)`

5) 20-minute downside slope
Take the closes of the last up to 20 completed bars including current bar.
Require at least 10 bars for this metric, else treat as 0.

Compute least-squares linear regression slope in PRICE units per bar using x = 0..n-1.
Convert to percent-per-bar relative to mean close of the window:

`slope_pct_per_bar = (slope_price_per_bar / mean_close_window) * 100`

Then:
`down_slope_20m_pct_per_bar = max(0, -slope_pct_per_bar)`

6) Range expansion on the last leg lower
Use:
- `short_range` = mean(high-low of the last 3 completed bars INCLUDING current bar)
- `base_range`  = mean(high-low of the 10 completed bars immediately BEFORE those 3)

`range_expansion = short_range / max(base_range, tiny)`

If insufficient history, degrade gracefully and do not alert until minimum history exists.

7) Volume expansion on the last leg lower
Use:
- `short_vol` = mean(volume of the last 3 completed bars INCLUDING current bar)
- `base_vol`  = mean(volume of the 10 completed bars immediately BEFORE those 3)

`volume_expansion = short_vol / max(base_vol, tiny)`

Final flush score
`flush_score` =
  25 * clip(drop_from_prior_30m_high_pct / 4.0, 0, 1) +
  20 * clip(distance_below_vwap_pct       / 2.0, 0, 1) +
  15 * clip(roc_5m_pct                    / 1.5, 0, 1) +
  10 * clip(roc_10m_pct                   / 2.5, 0, 1) +
  10 * clip(down_slope_20m_pct_per_bar    / 0.15, 0, 1) +
  10 * clip((range_expansion - 1.0)       / 1.5, 0, 1) +
  10 * clip((volume_expansion - 1.0)      / 2.0, 0, 1)

Clamp the final score to [0, 100].
Round for UI display to one decimal place.

Alert condition
Default alert when:
- flush detection is enabled
- current ET time is inside configured window
- enough history exists
- flush_score >= `flush.min_alert_score`
- symbol not on cooldown

Optional extra gating for v1 to reduce noise:
- if `require_below_vwap` is true, require `distance_below_vwap_pct > 0`
- if `require_drop_from_recent_high` is true, require `drop_from_prior_30m_high_pct > 0`
